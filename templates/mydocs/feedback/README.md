# `feedback/` 폴더 규칙

## 목적

자율 LOOP의 자기검토(self-review)·자기수정 기록과, 최종 게이트에서 들어온 인간 PR 리뷰 피드백을 보존한다. 다음 수정에서 놓치지 않도록 교정 입력을 남기는 폴더다.

## 답하는 질문

- "무엇이 charter 수용·검증 기준에 못 미쳤는가?"
- "어떻게 자기수정했는가, 또는 사람이 어떤 판단을 추가했는가?"

## 작성 시점

자기검증 MISS와 자기수정, 에스컬레이션 사유 기록, 최종 PR 인간 리뷰 피드백 수신 시점.

## 허용 파일명

- `{yyyymmdd}_{topic}.md`
- `task_{milestone}_{이슈번호}_feedback.md`

## 사용 템플릿

`mydocs/_templates/feedback.md`

## 반드시 포함할 내용

- 대상 이슈, Stage, PR
- 출처 (자기검증/자기수정, 에스컬레이션, 인간 PR 리뷰)
- 요약
- 상세 (MISS 원인 또는 리뷰 지적)
- 우선순위
- 반영 기준 (재검증 OK/MISS)
- 후속 확인

## 두면 안 되는 내용

- 기술 조사 전체
- 트러블슈팅 해결 과정 전체
- charter 자체를 바꾸는 결정 (charter급 사건은 에스컬레이션으로 처리)

## 다음 세션 AI가 복원해야 할 맥락

charter 기준 미달 원인과 반드시 반영해야 할 교정 기준.

# 에이전트 자율 실행과 울트라-워터폴 규율

## 목적

Claude와 Codex는 기본적으로 빠른 실행과 자율 수정에 최적화되어 있다. 울트라-워터폴은 이 자율성을 **활용**하되, charter 경계와 자기검증으로 규율한다. 위험은 자율성 자체가 아니라 **무절제한 자율**(charter 범위를 벗어나거나 검증을 건너뛰는 것)이다. 이 문서는 자율 실행 시 우선순위와 규율 기준을 정의한다.

## 주의 지점 (무절제한 자율의 징후)

1. charter 범위 이탈 — 요구되지 않은 리팩터링·범위 확장
2. 자기검증 생략하고 다음 Stage로 진행
3. 검증 MISS를 OK로 보고
4. charter 가정 붕괴·charter급 사건을 무시하고 자율 진행
5. 자기수정 N회 한도를 넘겨 무한 시도

## 우선순위 규칙

1. charter (인테이크에서 인간이 확정·잠금한 방향)
2. `AGENTS.md` 가드레일
3. task 문서(`plans`의 charter·구현계획, `working`, `report`, `orders`)
4. 에이전트 기본 동작

상위 규칙과 충돌하면 하위 규칙을 적용하지 않는다. charter로 판단 불가한 charter급 모호성은 인간에게 에스컬레이션한다.

## 운영 규칙

- 기본 원칙은 `인테이크(charter) -> 이슈·브랜치·오늘할일 -> 자율 LOOP(Stage: 구현 -> 자기검증 -> 기록) -> 최종 보고서 -> PR(인간 검토)` 순서다.
- 각 Stage 완료 후 charter 수용·검증 기준으로 OK/MISS를 자기검증한다. OK면 자동 진행, MISS면 같은 Stage 안에서 자기수정한다.
- 자기수정은 charter 한도 N까지만 시도한다. N회 실패 또는 charter급 사건이면 LOOP를 멈추고 인간에게 에스컬레이션한다.
- 이슈 close는 PR merge 이후에만 수행한다.
- 문서 수정은 기존 내용을 먼저 읽고 필요한 부분만 수정하며, 불가피할 때만 내용을 추가한다.
- 문서 위치/파일명 규칙을 위반한 산출물과 다음 작업에 필요하지 않은 로컬/원격 부산물은 즉시 정리한다.

## 실무 체크리스트

- 수정 전: charter가 `LOCKED` 상태인지, 변경이 charter 범위 안인지 확인
- 수정 중: 변경 범위가 charter 목표·포함 범위와 일치하는지, 비목표·제외·제약에 닿지 않는지 점검
- 수정 후: charter 검증 기준으로 OK/MISS 자기검증 실행, `_stage{N}.md` 기록, 종료/에스컬레이션 판정, `loop-state.json` 갱신

## 예외 처리

- 단순 문구 수정, 오탈자 수정처럼 위험이 매우 낮은 작업은 절차를 필요한 수준으로 축소할 수 있다.
- 단, charter의 목표·수용 기준은 축소하지 않으며, GitHub Issue가 열린 작업은 최소한 최종 보고서와 PR 본문에 결과를 남긴다.
